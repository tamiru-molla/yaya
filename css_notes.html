  4 - flexbox 
     - 1D layout system (row or columens)
     - container element (diplay : flex)
     - flex item (flex : flex_grow flex_shrink flex_basis)
                 ( flex : 1 0 auto)
        . Flex_grow - +ve space in parent conatiner
                    - (>= 0) - ( flex : 1  0  auto)
        . Flex_shrink - -ve space in parent cont
                      - (>= 0) - ( flex : 0 1 auto)
        . flex_basis  - item size b4 grow or shrink 
                      - assigning initial width for each
                        item before sharing available space
                        then saring will br according to the
                        proportion unit
                      - (auto, 0, size) - ( flex : 1 0 auto)
                      - [1 2] is same as [50 100] bn items
     ==> flex-flow : row wrap 
         - flex-direction : column (default : row) 
                       (row-reverse, column-reverse)
         - flex-wrap : wrap  with flex:width_px; 
                     - for fixed container size  overflow 
                       might be occur                
    * (diplay:inline-flex) -> (parent behave like inline)
                           -> ( child as flex item ) 
   ==> controling alignment (apply on flex container) 
    # align-items : flex-start/flex-end/center//baseline/streach(def)
        - column access alignment
        
    # justify-content : flex-start/flex-end/center/
                      space-around/space-between 
        - row access alignment
        
    # align-content :flex-start/flex/end/center 
                  space-between/space-around 
        - when there is >1 row (wraped items)
        - column access alignment 
        - apply on each row of items.
  
    # align-self and justify-self 
       - apply for each individual
    

  5) Grid - 2D layout system
    --> (@  grid container) 
    {
    display : grid  
    grid-template-columns : 1fr 1fr 1fr (3 col) 
                          : repeat(3, 1fr)
                          : minmax(10px,20px) 1fr 1fr
   - for dimention we can use minmax() mtd   
    --> minmax(min_size,max_size)  

    grid-template-rows : 1fr 2fr  (2 rows)
          - draw grid lines 
          - its distance bn each line (1 2 3 4) by 1 2 3 
          - fr (fraction - proportional to container spoace)
    grid-auto-rows : 100px; // if total # cell 
                           not enough for all grid items
    grid-column-gap : 2px;
    grid-row-gap : 2px;
    grid-gap : 2px; --> general form
     => then grid items populated from top-left to
        botom-right basd on HTML src order
    grid-auto-flow : column  // default is row
    } 
    
  --> grid item 
   to place each item  where we want  
     
-->  line based grid map mtd
          * asign start and end grid lines
          * time consumer
   @ grid item 
    {
      grid-column : 2/4  (2 / span 2) //same
                   bn line 2 and line 4 in coll lines
                  : 1 / -1  (-1 is end line )
      grid-row : 1/3 bn line 1 and line 4 in row lines
              : 5 (start at row 5) 
      order:1 (top left )
    }

  -->  text based grid map mtd
          * by assign area name to each cells  
          * by changing map order we can get d/t layout 
            for d/t devices  
          * Accessible mobile first layout work 
            well on all screen width
          
   @ grid container
   {
     grid-template-areas:
     " title title title "
     " main header header "
     " main footer footer ";

    ==>  ..... can  be used as empty cell
   }
   @grid item for each
   {
     .title {grid-area :[area-name]} // title

   }

   ==> to test for grid support use feature queries
    @support(display:grid) {...}
